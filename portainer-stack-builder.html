Portainer Stack Generator

Achievement Engine + ABS Stats

This builder outputs a single Portainer-compatible stack file for both containers and an optional user_xp_start.json sidecar. By default it generates a portable stack using Docker named volumes, with bind mounts available only when you need host-specific paths for Linux, macOS, Windows, or Unraid.

Use the generated stack in Portainer: Stacks -> Add stack -> Web editor.

          
    
            
  1. Portable mode uses Docker named volumes, so the generated YAML works across Linux, macOS, Windows, and Unraid without editing host paths.
    2. 
            
  2. Use bind-mount mode only if you need direct host file access. In that mode, replace the example paths with real paths for the Docker host running Portainer.
    3. 
            
  3. If you use XP start overrides, download user_xp_start.json. In named-volume mode, copy it into the achievement engine volume after deploy. In bind-mount mode, place it in the mapped data folder before or after deploy.
    4. 
            
  4. In Portainer, open Stacks, click Add stack, paste the generated YAML, and deploy.
    5. 
            
  5. After deploy, open the engine at http://host:dashboard-port/. If you need custom icons or a static override from the host filesystem, switch to bind-mount mode.
